Object Storage

Object storage adalah model penyimpanan data yang menyimpan informasi sebagai objek tunggal bersama dengan metadata dan sebuah identifier unik. Setiap objek dalam sistem penyimpanan objek memiliki alamat atau kunci yang unik, yang memungkinkan untuk mengaksesnya. Struktur penyimpanan ini berbeda dengan model penyimpanan blok atau berkas, yang masing-masing menyimpan data dalam format yang berbeda.

Beberapa karakteristik utama dari penyimpanan objek termasuk:

  1. Objek dan Metadata: Data dan metadata yang terkait disimpan bersama-sama sebagai objek tunggal. Metadata menyediakan informasi tentang karakteristik atau atribut objek, seperti tipe data, ukuran, dan tanggal pembuatan.
  2. Identifier Unik (Object ID atau Key): Setiap objek memiliki identifier unik atau kunci yang digunakan untuk mengaksesnya. Kunci ini biasanya dihasilkan berdasarkan hash dari data objek atau menggunakan mekanisme unik lainnya.
  3. Skalabilitas: Penyimpanan objek dirancang untuk mencapai skalabilitas yang tinggi. Ini memungkinkan penyimpanan dan pengelolaan sejumlah besar data dengan efisien.
  4. Distribusi Global: Sistem penyimpanan objek dapat diatur untuk mendukung distribusi global, memungkinkan akses ke objek dari berbagai lokasi di seluruh dunia.
  5. Tidak Ada Struktur Hirarkis: Tidak seperti sistem penyimpanan berkas yang menggunakan struktur hirarkis (folder dan subfolder), penyimpanan objek tidak memiliki struktur hirarkis. Setiap objek disimpan secara independen dan dapat diakses langsung menggunakan kunci atau identifier uniknya.

Penyimpanan objek umumnya digunakan untuk keperluan penyimpanan data besar seperti file media, arsip, dan data yang harus dapat diakses dengan skala besar dan dalam lingkungan distribusi. Beberapa penyedia layanan cloud menyediakan layanan penyimpanan objek untuk keperluan penyimpanan data di cloud computing.
